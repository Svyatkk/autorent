## Основний алгоритм

### 1. Бекенд: Підрахунок вільних днів ([CarService.php](./server/app/Services/CarService.php))
Алгоритм підрахунку вільних днів спрацьовує динамічно під час запиту списку автомобілів. Його головна задача - визначити, чи має машина вікно тривалістю не менше 9 годин (540 хвилин) у робочі години кожного дня місяця.

**Ключові константи сервісу:**
- `WORK_START_HOUR = 9` (початок робочого дня - 09:00)
- `WORK_END_HOUR = 21` (кінець робочого дня - 21:00)
- `MIN_FREE_MINUTES = 540` (мінімальна тривалість вільного вікна - 9 годин)

**Потік виконання функцій:**

- `appendAvailability(Car $car, Carbon $from, Carbon $to)`
Це вхідна точка алгоритму. Функція отримує об'єкт авто та межі місяця. Вона встановлює дві динамічні властивості для відповіді: `all_days` (кількість днів у місяці) та `free_days` (яка розраховується через виклик `countFreeDays`).

- `countFreeDays(Collection $bookings, Carbon $from, Carbon $to)`
Функція-цикл. Вона ітерує кожен день вибраного місяця (від `$from` до `$to`). Для кожного дня викликає метод `isCarFreeForDay()`. Якщо метод повертає `true`, лічильник вільних днів збільшується на 1.

- `isCarFreeForDay(Collection $bookings, Carbon $day)`
Основне ядро логіки. Визначає, чи є достатньо вільного часу в конкретний день:
  - Встановлює межі для перевірки: `dayStart` (09:00) та `dayEnd` (21:00).
  - Збирає всі інтервали бронювань, які перетинаються з цим робочим днем, викликаючи `getBusyIntervalsForDay()`. Нормалізує ці інтервали, об'єднуючи ті, що накладаються один на одного (через `mergeBusyIntervals()`).
  - Проходить по об'єднаних зайнятих блоках і шукає проміжки між ними (або між 09:00 та першим блоком, або між останнім блоком та 21:00). Якщо знайдено проміжок `>= 540` хвилин, функція негайно повертає `true`.

**Допоміжні функції нормалізації:**
- `getBusyIntervalsForDay()`: Відсікає бронювання, які не стосуються поточного дня, та обрізає краї бронювань так, щоб вони не виходили за межі 09:00 - 21:00.
- `mergeBusyIntervals()`: Сортує отримані інтервали за часом початку та зливає (merge) пересічні інтервали в суцільні блоки зайнятості, щоб уникнути подвійного підрахунку часу.

### 2. Фронтенд: Рендеринг сітки календаря ([CalendarGrid.tsx](./client/src/components/CalendarGrid/index.tsx))
Візуалізація календаря побудована на принципі абсолютно позиційованих блоків всередині відносного контейнера рядка таблиці. Позиція і ширина кожного бронювання вираховуються математично на основі фіксованої ширини комірки дня.

**Ключові константи компонента:**
- `CELL_WIDTH = 36` - кількість пікселів, що відповідає рівно 1 добі.
- `DAY_MS = 86400000` - кількість мілісекунд у добі (для переведення дат у дні).

**Як працює рендеринг:**
Під час генерації рядка для автомобіля, компонент мапить масив `car.bookings`. Для кожного бронювання зі статусом 1 (Rented) викликається головна розрахункова функція - `getBookingStyle()`.

**Логіка `getBookingStyle(booking, year, month)`:**
- **Парсинг та валідація:** Функція отримує дати старту та кінця бронювання у форматі рядка та конвертує їх у JS-об'єкти Date через допоміжну функцію `parseBookingDate()`. Якщо бронювання не потрапляє у вибраний користувачем місяць (`monthStart` - `monthEnd`), функція повертає `null` і блок не рендериться.
- **Обрізка по межах місяця (`visibleStart`, `visibleEnd`):** Якщо бронювання почалося минулого місяця або закінчується наступного, його краї обрізаються до першого/останнього числа поточного місяця. Це робиться для того, щоб синій блок не вилазив за межі видимої сітки календаря.
- **Розрахунок відступу (`leftDays` -> `left`):** Система рахує різницю у мілісекундах між початком місяця (`monthStart`) та видимим початком бронювання (`visibleStart`), після чого ділить це на `DAY_MS`. Результат: кількість днів (включно з дробовою частиною) від початку сітки. Це значення множиться на `CELL_WIDTH`, формуючи CSS-властивість `left` (наприклад, `left: 180px`).
- **Розрахунок ширини (`widthDays` -> `width`):** Система рахує різницю між видимим кінцем та видимим початком бронювання, переводить у дні та множить на `CELL_WIDTH`. Також застосовується `Math.max(..., 4)`: це гарантує, що навіть надзвичайно короткі бронювання матимуть мінімальну ширину 4px і будуть помітні на UI. Формується CSS-властивість `width`.

**Фінальний результат:** Розрахований об'єкт (напр. `{ left: '100px', width: '72px' }`) передається як `style` у `<div className={styles.bookingBar}>`, який має `position: absolute`, що дозволяє розмістити блок над відповідними датами.

## Запуск проєкту

### Вимоги

- [Docker Desktop](https://www.docker.com/products/docker-desktop/) встановлений і запущений
### Кроки

**1. Клонувати репозиторій**
```bash
git clone <url-репозиторію>
cd autorent
```
**2. Запустити всі сервіси**
```bash
docker compose up -d --build
```

Перший запуск займає 2–5 хвилин. MySQL автоматично імпортує `db.sql` при першому старті.

**3. Відкрити застосунок**

| Сервіс | URL |
|---|---|
| Фронтенд | http://localhost:3000 |
| API (Laravel) | http://localhost:8000/api |
| phpMyAdmin | http://localhost:8080 |

Логін phpMyAdmin: `root` / `root`
